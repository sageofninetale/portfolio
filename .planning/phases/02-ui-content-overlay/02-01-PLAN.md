# Phase 2: UI Content Overlay & Motion Plan

## Context
The hero section has been updated to a premium Web3 dark aesthetic with a static looping video background. The user wants the rest of the portfolio (About, Education, Work, Contact) to precisely match this design language. Furthermore, the user explicitly requested the return of the "Tunnel effect / camera motion" feeling. Since we are no longer using WebGL for the background, we will simulate this immersive 3D flight experience by applying deep GSAP/Scroll-driven scale and 3D transform animations to all DOM content blocks as they enter the viewport.

## 3. Implement the "Tunnel" 3D Scroll Motion
Rebuild the feeling of diving through a space vortex by animating the DOM sections dynamically on scroll.

<task>
<read_first>script.js</read_first>
<action>
1. Register GSAP ScrollTrigger: `gsap.registerPlugin(ScrollTrigger);`
2. Select all `.tunnel-section` elements.
3. Create a GSAP timeline for each section where the initial state is `scale: 0.5, opacity: 0, z: -500` (deep in the background).
4. As the section enters the viewport (`start: "top 90%"`), animate it to `scale: 1, opacity: 1, z: 0`.
5. Apply a slight exiting animation (scrub: true) so as it leaves the top of the screen (`end: "bottom top"`), it scales up slightly and fades out (`scale: 1.2, opacity: 0`), perfectly simulating the effect of the camera physically moving through a tunnel past the elements.
</action>
<acceptance_criteria>
- script.js contains a GSAP ScrollTrigger loop iterating over sections.
- The animation strictly changes `scale` and `opacity` based on scroll position to emulate a camera pushing forward.
</acceptance_criteria>
</task>
